# 📅 Веб-приложение "График Дежурств"

Современное веб-приложение на C++ для автоматического управления графиком дежурств и распределения задач.

## Возможности

- ✅ **Управление задачами** - создание задач с настраиваемой периодичностью (например, уборка ванны каждые 3 дня)
- 👥 **Управление пользователями** - добавление и редактирование пользователей
- 📋 **Автоматическое распределение** - справедливое распределение дежурств между пользователями
- 📊 **Статистика** - отслеживание выполненных и пропущенных дежурств

## 🛠️ Требования

- CMake 3.15 или выше
- Компилятор C++17 (GCC, Clang или MSVC)


## 📦 Сборка проекта

### Linux/macOS

```bash
# Создайте директорию для сборки
mkdir build
cd build

# Запустите CMake
cmake ..

# Соберите проект
make

# Запустите приложение
./DutyScheduleApp
```

### Windows

```bash
# Создайте директорию для сборки
mkdir build
cd build

# Запустите CMake
cmake ..

# Соберите проект
cmake --build . --config Release

# Запустите приложение
Release\DutyScheduleApp.exe
```

## 🚀 Использование

### Запуск приложения

**Простой способ:**
```bash
./run.sh
```

**Или вручную:**
```bash
cd build
./DutyScheduleApp
```

После запуска:
1. Сервер запустится на порту 8080
2. Откройте браузер и перейдите по адресу: `http://localhost:8080`
3. Добавьте задачи (например, "Уборка ванны", "Мытье посуды")
4. Добавьте пользователей (или используйте существующих)
5. Нажмите "Сгенерировать расписание" для автоматического создания графика дежурств


## 📖 API Endpoints

### Задачи
- `GET /api/tasks` - получить все задачи
- `POST /api/tasks` - создать новую задачу
- `DELETE /api/tasks/:id` - удалить задачу

### Пользователи
- `GET /api/users` - получить всех пользователей
- `POST /api/users` - создать нового пользователя
- `PUT /api/users/:id` - обновить пользователя
- `DELETE /api/users/:id` - удалить пользователя

### Дежурства
- `GET /api/duties` - получить дежурства (с опциональными параметрами startDate, endDate, userId)
- `POST /api/duties/generate` - сгенерировать расписание
- `POST /api/duties/:id/complete` - отметить дежурство как выполненное
- `POST /api/duties/:id/skip` - пропустить дежурство
- `DELETE /api/duties/:id` - удалить дежурство

### Статистика
- `GET /api/statistics` - получить статистику (с опциональными параметрами startDate, endDate)
